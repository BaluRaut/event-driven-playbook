# Event-Driven Architecture Pro Playbook

**16 topics** on the messaging half of distributed systems — how services talk without falling over. Each topic is taught five ways so it sticks: **What/Why/How**, an **ASCII diagram**, **two analogies**, real **code**, an **"In plain words"** 8th-grade version, plus a **Do/Don't** and the **gotcha that bites in production**.

🔗 **Live:** https://baluraut.github.io/event-driven-playbook/

## What's inside

**Part I — Messaging Foundations (01–08):** sync vs async, message queues vs event streams, pub/sub & fan-out, Kafka fundamentals (topics/partitions/offsets), delivery semantics, idempotent consumers, ordering & partitioning, choreography vs orchestration.

**Part II — Event-Driven Patterns (09–13):** event sourcing, CQRS, the Saga pattern, the outbox pattern, dead letter queues & poison messages.

**Part III — Production Streaming (14–16):** schema evolution & the registry, backpressure/consumer lag/replay, and a **capstone: an event-driven order system.**

## Companion

The **machinery** underneath — CAP, consistency, consensus, replication, sharding, caching, resilience — lives in the companion [Distributed Systems Playbook](https://baluraut.github.io/distributed-systems-playbook/).

Includes a **round-based quiz** (5 per round, 16 total) that **saves progress in the browser**.
